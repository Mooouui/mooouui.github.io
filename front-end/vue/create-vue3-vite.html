配置 eslint

安装 eslint

yarn add -D eslint

eslint init

   yarn eslint --init

result -

会在目录下生成.eslintrc.*文件，但会有一个报错，需要在 env 下加"node": true

// ❌ 'module' is not defined.
module.exports = {
  env: {
    browser: true,
    es2021: true,
    // add here resolve *'module' is not defined* bug
    node: true,
  },
  // ...
};

eslint 对 env node 的解释：node - Node.js 的全局变量和 Node.js 的范围。 它会暴露出 nodejs 的全局变量（module 是其中之一）

在 package.json 配置 script

在 script 下添加"lint": "eslint . --ext .vue,.js,.ts,.jsx,.tsx --fix"

# 运行
yarn lint

会提示有报错信息

moui@bogon vite-project % yarn lint
yarn run v1.22.19
$ eslint . --ext .vue,.js,.ts,.jsx,.tsx --fix

/Users/moui/CodeSpace/vite-project/src/App.vue
  1:8  error  Parsing error: '>' expected

/Users/moui/CodeSpace/vite-project/src/components/HelloWorld.vue
  1:8  error  Parsing error: '>' expected

✖ 2 problems (2 errors, 0 warnings)

error Command failed with exit code 1.

TIP

原因：eslint 配置的 parser: "@typescript-eslint/parser" 是不会处理 vue 的 SFC 文件的 lint 的，对于.vue 的文件需要借助 eslint-plugin-vue，在eslint-plugin-vue的官方文档文档中有说明：如果您已经使用另一个解析器（例如 "parser": "@typescript-eslint/parser" ），请将其移至 parserOptions 中，这样它就不会与此插件配置使用的 vue-eslint-parser 冲突 所以要把 eslint 的配置文件改为：

- "parser": "@typescript-eslint/parser",
+ "parser": "vue-eslint-parser",
  "parserOptions": {
+     "parser": "@typescript-eslint/parser",
  }

配置 prettier

安装 prettier

yarn add -D prettier

创建配置文件

touch .prettierrc.cjs

内容配置参考：

module.exports = {
  // 一行的字符数，如果超过会进行换行，默认为80
  printWidth: 80,
  // 一个tab代表几个空格数，默认为2
  tabWidth: 2,
  // 是否使用tab进行缩进，默认为false，表示用空格进行缩减
  useTabs: false,
  // 字符串是否使用单引号，默认为false，使用双引号
  singleQuote: true,
  // 行位是否使用分号，默认为true
  semi: false,
  // 是否使用尾逗号，有三个可选值"<none|es5|all>"
  trailingComma: "none",
  // 对象大括号直接是否有空格，默认为true，效果：{ foo: bar }
  bracketSpacing: true,
};

解决冲突

理论上eslint用于解决我们的代码规范，确保我们的代码质量， 而prettier则是用于优化我们的代码格式，提供更好的可读性和美观度。 即使用 Prettier 来解决代码格式问题，使用 linter 来解决代码质量问题 但实际上 Linters 通常不仅包含代码质量规则，还包含风格规则。使用 Prettier 时，大多数风格规则都是不必要的，但更糟糕的是——它们可能与 Prettier 发生冲突！ 但幸运的是可以通过一些配置关闭 linters 和 prettier 的冲突

  1. eslint-config-prettier 关闭所有不必要或可能与 Prettier 冲突的规则。

  2. eslint-plugin-prettier 以 ESLint 规则运行 Prettier，并将差异报告为单独的 ESLint 问题 （基于 prettier 代码风格的 eslint 规则，即 eslint 使用 pretter 规则来格式化代码。）

TIP

eslint-plugin-prettier: This plugin works best if you disable all other ESLint rules relating to code formatting, and only enable rules that detect potential bugs. (If another active ESLint rule disagrees with prettier about how code should be formatted, it will be impossible to avoid lint errors.) You can use eslint-config-prettier to disable all formatting-related ESLint rules.

所以基于此，我们可以配置

yarn add -D eslint-config-prettier eslint-plugin-prettier

TIP

插件eslint-plugin-prettier附带一个 plugin:prettier/recommended 配置，可一次性设置插件和 eslint-config-prettier

然后您需要添加 plugin:prettier/recommended 作为 .eslintrc.* 中的最后一个扩展：

module.exports = {
  //...
  extends: [
  'eslint:recommended',
  'plugin:vue/vue3-essential',
  'plugin:@typescript-eslint/recommended',
+ 'plugin:prettier/recommended' // 新增，必须放在最后面
  ],
  //...
}

之后我们就可以在.prettierrc.*中设置自己的 prettier 规则了
