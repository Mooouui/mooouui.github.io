JS 概念深入理解碎片

类型转换

隐式转换

  • 当不同类型的值进行比较的时候，JS 会先将其转化为数字（number）再判断大小
  • 当在if(...)语句中判断的时候 会将括号内的表达式的计算结果隐式转换为 boolean 值

案例：

// 1
let a;
if ("  ") {
  a = 5;
}
console.log(a); // undefined

// 2
let b;
if (" " == true) {
  b = 5;
}
console.log(b); // 5

原理：

“ ” 被称为空白（非空字符串），它在转换为 boolean 值时候是 true, 但是转换为 number 的时候是 0

强制转换（显式转换）

  • 值得注意的是：

    • 对 undefined 进行数字型转换时，输出结果为 NaN，而非 0。
    • 对 "0" 和只有空格的字符串（比如：" "）进行布尔型转换时，输出结果为 true。

== 和 === 和 >= ...

  1. 普通的相等性检查==存在的问题

    • 它不能区分出 0 和 false
    • 无法区分空字符串""和 false
alert(0 == false); // true

alert("" == false); // true

  1. 严格相等运算符===

    严格相等运算符 === 在进行比较时不会做任何的类型转换。

  2. 对 null 和 undefined 比较

    1. 当 === 比较， 他们不相同
    2. 当 == 比较， 他们相同

  3. null vs 0

alert(null > 0); // false
alert(null == 0); // false
alert(null >= 0); // true

why?

  • 相等性检查 == 和普通比较符 > < >= <= 的代码逻辑是相互独立的

    undefined 和 null 在相等性检查 == 中不会进行任何的类型转换 而在普通比较 中， null 和 undefined 会先被转换为数字

空值合并运算符 ??

定义：如果第一个参数不是 null/undefined，则 ?? 返回第一个参数。否则，返回第二个参数

常用场景：提供默认值

alert(a ?? "anonymous"); // anonymous

const b = "alan";
alert(b ?? "anonymous"); // alan

a ?? b 的结果是

如果 a 已定义，返回 a， 否则返回 b
